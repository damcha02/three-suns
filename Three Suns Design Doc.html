Sun Personalities

Each sun has a distinct character — motion style, halo color, corona behavior. The player learns to read them subconsciously over time.

A

Sun Alpha — The Tyrant

Largest. Moves with slow gravitational authority. Its corona pulses in amber-red. When it drifts close, the planet's day side starts to glow. Three.js: PointLight tinted 0xff4400, intensity ramps 1→4 on proximity. Halo: additive-blended disc mesh, scale oscillates 1.0→1.15 at 0.3Hz.

B

Sun Beta — The Ghost

Smallest, fastest, erratic elliptical orbit. Blue-white. Its light is cold. When it rules the sky, temperatures drop — blue tint bleeds into skybox. Three.js: PointLight tinted 0x88ccff. Trail: 12-point ribbon of additive-blended PlaneGeometry quads fading opacity 0.4→0. Ghost-like flicker every 3–7s (intensity ×0.4 for 80ms).

FROZENFAST ORBITUNSTABLE
C

Sun Gamma — The Stranger

Medium mass, chaotic attractor. Its orbit shifts unpredictably — it seems to "notice" the planet. Violet-magenta. When it achieves alignment with the other two, a brief gravitational resonance occurs (the worst outcome). Three.js: PointLight tinted 0x9955ff. Shader: subtle chromatic ring rendered as TorusGeometry, rotation speed randomly reverses.

Planet & Starfield

Earth-Like Planet

SphereGeometry, radius ~1.5. Material: MeshPhongMaterial with emissive 0x001133 (city lights on dark side, always slightly glowing). City light texture: procedural — tiny bright dots scattered with noise. Atmosphere ring: additive-blended TorusGeometry, tinted by dominant sun color. When dying: emissive fades to 0 (frozen) or spikes to orange (burned). The planet should never look fully dead until the death screen appears.

Starfield / Nebula

Three layers: (1) Static deep-field — BufferGeometry of ~3000 points, white, opacity 0.6. (2) Mid nebula — two large PlaneGeometry quads, additive blend, wispy color (deep violet + dark teal), opacity 0.04–0.08. (3) Danger reaction layer — when omen triggers, a faint red/orange wash grows behind everything (fog density increase). Stars never twinkle in safe state; on danger they jitter position ±0.5px via a vertex shader uniform uJitter ramping 0→1.

Death Transition — Sequence

→ 0ms — Death detected. Game time stops. CSS overlay begins fade-in from opacity 0.
→ 0–400ms — Overlay reaches opacity 1. Starfield dims to 25%. Background tint appears as radial gradient from bottom.
→ 400ms — Planet fades to ghost state (opacity 0.06) and drifts 20px downward (CSS transform transition).
→ 500ms — Death title appears: scale(0.96) opacity(0) → scale(1) opacity(1), 600ms ease-out-expo.
→ 900ms — Subtitle fades in. Cause text fades in at 1100ms.
→ 1200ms — Buttons appear. Keyboard hint appears last at 1400ms.
→ No sound effects — silence is part of the horror. Optional: a single very low-frequency tone at 0ms, 0.2s duration, volume 0.05.

04 Gameplay Readability

Communicating Danger Without Clutter

Danger — Screen-Edge Heat

As a sun approaches danger proximity, a colored vignette grows inward from the screen edges. Burned → red-orange. Frozen → deep blue. The vignette is a pure CSS box-shadow: inset 0 0 200px COLOR on a fixed overlay div. Opacity ramps 0→0.35 over 30 seconds before the threshold. No sudden flashes. Players feel it before they see it.

◈

Omens — Glitch Text Intrusion

When an omen triggers, a single line of monospace text appears at top-center, letter-spacing wide, opacity 0.7, 12px. It fades in over 500ms, stays 4s, fades out over 1s. Use CSS @keyframes to slightly distort letter-spacing during fade-in (0.5em→0.25em). Examples: "⚠ ALPHA APPROACH · 12 DAYS" or "RESONANCE BUILDING · 3%". Never more than one omen line at once. Color matches the relevant sun.

Gravity Anchors — Quiet Presence

Anchors appear as faint crosshair markers — a thin ring (TorusGeometry, wireframe, radius 0.3) with four radiating lines (thin LineSegments). Color: dark gold (collapse palette). They pulse scale 1.0→1.05 at 0.5Hz — alive but not distracting. When the player targets one, outline brightens. No fill, no glow until activation. Activated: inner ring collapses inward, opacity spikes to 1 for 200ms, then settles.

Focus Pulse — Rings of Clarity

On activation: three concentric rings emit from the planet center (CSS absolute divs or Three.js ring meshes). They scale outward from 0 to 3× over 800ms, fade opacity 1→0. Color: match current stability color (cool violet = safe, warm orange = stressed). The rings are the visual "acknowledgment" — the player knows the action registered. No sound needed. Cheap: three divs, animation: pulse-ring 0.8s ease-out forwards.

Three Survival Modes — Visual Identity

EQUILIBRIUM

Default. Planet atmosphere is calm blue-green. Stability bar is violet. Mode label outlined. Background nebula cool-toned.

HIBERNATION

Planet dims, emissive drops 40%. City lights fade slightly. Atmosphere gains blue tint. Mode label turns frozen blue — suggests deliberately going dark to survive cold.

SACRIFICE

Planet atmosphere reddens. Emissive spikes warm. Mode label turns amber. Screen-edge vignette burns slightly — you're choosing controlled burn over uncontrolled. Risky, readable.

05 Juice List

10 Effects That Make It Feel Premium

Each effect is achievable in under 30 minutes. Each amplifies the existing game state rather than adding new systems.

01
Screen-Edge Vignette Danger
A fixed CSS div with box-shadow: inset 0 0 180px COLOR, pointer-events:none. Opacity driven by nearest sun's proximity. Zero Three.js overhead. Makes every monitor edge feel like a warning.
~15 MIN · PURE CSS
02
Camera Micro-Shake on Death
On death trigger: camera.position.x += Math.sin(t * 40) * 0.05 * decay for 600ms where decay goes 1→0. Not a screenshake — a gentle tremor, like the planet is sighing. Stops before death screen appears.
~20 MIN · 5 LINES JS
03
Star Jitter on Danger
BufferGeometry star positions get a uJitter uniform (0→1) passed to vertex shader. Stars wiggle ±0.3 units at high danger. When safe, jitter returns to 0 over 3s. Makes the entire universe feel unstable. The background itself becomes a readability signal.
~30 MIN · VERTEX SHADER
04
Day Number Increment Pop
Every new day, the day counter scales 1.0→1.18→1.0 over 300ms (CSS keyframe). A micro-event the player notices without being told. Tiny but makes the score feel earned. On milestone days (50, 100, 200…) add a brief golden glow.
~10 MIN · CSS
05
Focus Pulse Ring Emission
Three CSS divs, border-radius 50%, positioned absolute over canvas center. On focus action: animate from scale(0.1) opacity(0.8) to scale(3) opacity(0), staggered 0/150/300ms. Color matches current mode. Costs nothing but a DOM append and CSS animation.
~15 MIN · CSS ANIMATION
06
Sun Halo Breathing
Each sun has a PlaneGeometry disc (additive blend, radial gradient texture drawn once to canvas). Scale oscillates independently: Alpha at 0.28Hz, Beta at 0.47Hz, Gamma at 0.33Hz. Slight size variation makes them feel alive. No animation loop change needed — multiply by sin(time × freq) in existing update loop.
~20 MIN · THREE.JS
07
City Lights Fade-to-Dark on Death Approach
Planet emissive map has city light dots. As stability drops below 20%, emissive intensity lerps toward 0. The planet literally going dark as civilization fails is a visceral signal that needs no UI. Costs: one mesh.material.emissiveIntensity = lerp(1, 0, stress).
~10 MIN · 1 LINE
08
Omen Text Letter-Spacing Entrance
Omen CSS div animates: letter-spacing from 0.8em → 0.25em over 400ms as it fades in. This creates a "closing in" effect — the text gathers itself as if materializing from noise. Done with a single @keyframes. Makes the omen feel different from all other UI elements.
~10 MIN · CSS
09
Death Title Scale-In
Death title starts at scale(0.94) opacity(0). Eases to scale(1) opacity(1) with cubic-bezier(0.16, 1, 0.3, 1) (expo-out) over 700ms. This single animation makes the death screen feel like a cinematic reveal instead of a popup. Zero JS needed — pure CSS transition on class add.
~5 MIN · CSS
10
High Score Moment — Gold Pulse
When player surpasses their best score: the day counter briefly turns oklch(68% 0.13 62) (dark gold) with a 400ms glow pulse, then returns to white. No text, no popup — just a color shift the attentive player notices. On death screen, if it was a new high score, the subtitle line gains a gold asterisk.
~15 MIN · CSS + JS
06 Implementation Notes for Claude Code

Concise Change List

Prioritized by impact. Avoid rewriting game logic — all changes are additive overlays and style passes.

FONTS & BASE STYLES
  • Add Google Fonts import for Syne (800) + Space Mono (400,700) in <head>
  • Set body { background: #06060e; font-family: 'Space Mono', monospace; }
  • Wrap entire Three.js canvas in a #game-root div; all overlay divs are siblings inside it
DEATH SCREEN OVERLAY
  • Create #death-overlay — position:fixed; inset:0; z-index:100; display:none; — shown on death event
  • Death overlay background: radial-gradient(ellipse at 50% 120%, VAR_TINT/0.2 0%, #06060e 60%) — set VAR_TINT via JS class on overlay: .burned | .frozen | .chaos | .collapse
  • Stars div (position:absolute, background-image radial-gradient dots, opacity:0.25) inside overlay — static, no animation
  • Planet ghost div: position:absolute; bottom:8%; left:50%; transform:translateX(-50%); width:15vmin; height:15vmin; border-radius:50%; background: radial-gradient(circle, TINT/0.15, transparent 70%);
  • Death title: font-family:'Syne'; font-weight:800; font-size:clamp(36px,7vw,96px); line-height:0.9; letter-spacing:-0.02em; — color = CSS var per death type
  • On show: add class .visible to overlay; CSS transition on opacity 0→1 (400ms). Title gets separate class .animate-in: transform:scale(0.94)→scale(1); opacity:0→1; transition: 700ms cubic-bezier(0.16,1,0.3,1); delayed 400ms
  • Subtitle: font-family:'Space Mono'; font-size:clamp(10px,1.2vw,14px); letter-spacing:0.2em; opacity:0.45; text-transform:uppercase;
  • Cause text: Space Mono 12px, opacity 0.3, not all-caps — injected dynamically by cause type
  • Buttons: font-family:'Space Mono'; font-size:11px; letter-spacing:0.18em; padding:14px 28px; background:transparent; border:1px solid TINT/0.4; border-radius:2px; cursor:pointer; — PLAY AGAIN uses tint color, others use rgba(255,255,255,0.25)
  • Stagger button appearance: each button has animation-delay: 1200ms, 1300ms, 1400ms fade-in
  • COPY SCORE button: copies formatted string to clipboard via navigator.clipboard.writeText()
HUD
  • Day counter: #hud-day — Syne 800, 36px, top-left. On day increment: add class .pop → CSS keyframe scale(1)→scale(1.18)→scale(1) 300ms, then remove class
  • Stability bar: #hud-stability — 80px wide, 2px tall, border-radius:1px. Inner fill width driven by stability%. Color transitions burned/frozen/chaos/collapse via JS style update
  • Three sun dots (top-center): fixed-size colored circles, opacity driven by sun threat level (0.3 safe → 1.0 critical). Add subtle glow box-shadow that scales with threat
  • Mode buttons (desktop: text labels at bottom-left): active mode has border-color 0.25, others 0.06. No background fill on any mode button
  • Vibe Jam portal: bottom-right, 10px Space Mono, opacity 0.2, pulsing dot, <a> tag wrapping it
  • Omen text: #hud-omen — top-center, absolute, initially opacity:0. Show: animate letter-spacing 0.6em→0.2em + opacity 0→0.75 over 400ms. Auto-hide after 4s
DANGER VIGNETTE
  • Add #vignette — position:fixed; inset:0; pointer-events:none; z-index:50; border-radius:0; transition: box-shadow 1s ease;
  • In game loop: compute dangerColor from nearest threatening sun and dangerAlpha 0→0.35. Set vignette.style.boxShadow = `inset 0 0 200px rgba(R,G,B,${dangerAlpha})`
  • Reset to no box-shadow when danger = 0. That's the entire system.
THREE.JS SCENE
  • Sun halos: for each sun, create PlaneGeometry(4,4) with MeshBasicMaterial, map=canvas texture (radial gradient drawn once), blending=AdditiveBlending, depthWrite:false. Scale in update loop: halo.scale.setScalar(1 + 0.12 * Math.sin(t * freq))
  • Planet emissive: planetMat.emissiveIntensity = THREE.MathUtils.lerp(1, 0, stressLevel) each frame. stressLevel 0(safe)→1(critical)
  • Star jitter: pass uJitter uniform to star vertex shader. In shader: vec3 pos = position + vec3(sin(position.y*10.0)*uJitter, cos(position.x*10.0)*uJitter, 0.0) * 0.3;
  • Anchor crosshairs: TorusGeometry(0.25,0.01,8,32) wireframe + 4 LineSegments (short arms). MeshBasicMaterial color 0xc49a30, opacity 0.4. Scale oscillation: 1 + 0.04 * Math.sin(t * 3.2)
  • Focus pulse: CSS rings only (no Three.js mesh needed). Emit 3 divs from #pulse-container absolutely centered over canvas. Reuse with an object pool of 3 ring divs.
DEATH TRANSITION SEQUENCE
  • On death: (1) set game state to DEAD, stop physics/time, (2) 0ms: show overlay div, start opacity transition, (3) 400ms setTimeout: planet ghost div appears, title animates in, (4) 900ms: subtitle, (5) 1100ms: cause text, (6) 1200ms: buttons stagger in
  • All timings via setTimeout chains or CSS animation-delay — no animation library needed
  • On PLAY AGAIN: hide overlay immediately (opacity:0, display:none after 300ms), reset game state, restart loop
PERFORMANCE NOTES
  • All overlays use will-change: opacity, transform only when animating; remove after animation completes
  • Vignette box-shadow is GPU-composited; safe to update every frame via RAF
  • Sun halo canvas texture: draw once at init (64×64 px), reuse. Never recreate per-frame.
  • Focus pulse rings: pool 3 divs, toggle classes — never appendChild during gameplay
  • Omen text: single div, innerHTML replace — no DOM creation on trigger
